# Code styles

## General

- No spelling mistakes
- Files should end with a trailing newline

## Java files

- The Java code follows the standard Java code style.
- The code should not contain unused imports.
- The code should not contain wildcard imports.
- Use idiomatic Java code, that fits the capabilities of the used Java JDK version. 

## JSON files

- JSON files are formatted with an indent of 2.
- JSON files should use trailing brackets in a new line for objects and arrays.

## Markdown files

- Markdown files follow the general Markdown syntax.
